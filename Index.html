Basics

Html (or hypertext markup language) is a language developed for the World Wide Web to build out the foundation of a website. To build out structure html uses markup in the form of elements to specify basic ideas within the html hierarchy. Such elements look like this:

<html> <head><body><header><div>

Within html the process of writing these elements out is not constricted by case-sensitivity like with other languages. As an example, the element <div> could be written like <Div> or <DIV> and it would mean the same thing regardless of circumstances.

How elements work

Elements can be thought of as the individual snip-bits of code that complete a certain task. It is like how different components are needed to build a house, an example being:

  • windows
  • concrete
  • insulation
  • Plumbing
  • Electric
  • AC and Heating unit
  • Planning

The basic format of an element begins with an opening statement and ends with a closing statement having the information to display in the middle like this: <p>Some piece of text<p> The type of tag that is used will depend on the needs of the situation. If there is a need for a picture the <img> tag would be used which would display a photo of the developer's choosing. This type of element does not use any text in order to work but simply a link to where the image is stored: <img src="Users/Name/Documents">

Attributes

One of the major advantages of html is that it is incredibly flexible due to its ability to be styled with other languages like css. The way that this is possible is through attributes that connect the styling sheet to the html like class and id. There are other attributes as well that do different things like src, href, target, and others. The src attribute for example handles where the element receives information like in the above example: <img src="Users/Name/Documents"> At the end of the day, attributes can be thought of as the modifiers to existing elements that help them do different tasks.

Structure of an html document

Overall, an html document needs a variety of elements and attributes to keep it running. It is akin to how the body operates without certain organs or muscles the body can operate but not as well. The basic structure of an html document firstly needs a signal telling the browser what version of html the website is using and that is where doctype comes in. The most recent one that works well for modern projects is <!DOCTYPE html> which should be at the start of all your html documents until a new standard comes out.

Next up on the list is the html element which houses every other element within the html document. The html tag carries both the head and body elements which makes up the entirety of the document.

The head

The job of the head element is to apply the rules wanted for the file as well as the links for styling. It also holds meta, title, and link elements.

The meta element oversees all data that isn’t associated with the other tags: <base> <link> <script> <style> <title> The average html document has the tag <meta charset=”utf-8"> which, “includes most characters from the vast majority of human written languages.”

The title element is in charge of the title of the webpage in the web browser.

The body

Within the body element confines the rest of the document and everything that is visible to the viewer of the website. This can include videos, images, text, and anything else you would like to have on a webpage. The body element works the same as any other element <body>rest of document</body>
